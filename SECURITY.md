# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please **do not** open a public issue.

Instead, report it privately to the project maintainers via email or by creating a **GitHub Security Advisory** (if enabled on the repository).

Please include:
- A description of the vulnerability
- Steps to reproduce
- Potential impact

We will acknowledge receipt within 48 hours and work on a fix before public disclosure.

## Best Practices

- Never hardcode credentials, project IDs, or API keys in scripts
- Use Application Default Credentials (ADC) or environment variables for authentication
- Regularly audit IAM permissions and service account keys
- Follow the principle of least privilege
